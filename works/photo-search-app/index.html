画像検索アプリ
（WebAPIを利用した動的アプリケーション）

画像検索アプリです。
世界中の写真が集まっている「Pexels」というサイトのAPIを使って、キーワードを入力すると、そのキーワードに合った写真を表示します。「もっと見る」ボタンを押すことで、ページを移動せずに写真を次々と読み込むことができます。

サイトをみる

要件

  • 外部APIからデータを取得し、動的に表示・操作できるアプリを作成
  • Vite + React Router、またはNext.js
  • エラーハンドリング（API取得失敗時の表示）の実装
  • Loading状態（データ取得中のぐるぐるなど）の実装

使用技術

React / API / Git / GitHub

工夫点

ユーザー体験（UX）を意識したUI設計
API通信の待ち時間に「読み込み中...」というローディング表示を追加し、ユーザーがストレスを感じないようにしました。また、通信エラーが発生した場合も、ただ動かなくなるのではなく、適切なエラーメッセージを表示するように実装しました。

「もっと見る」機能
ページネーション（1ページ、2ページ...）ではなく、既存のリストの下に新しい写真を追加していく方式を採用しました。これにより、ユーザーはスムーズに多くの写真を閲覧できます。

環境変数によるセキュリティ管理
APIキーをソースコードに直接記述せず、.envファイルとViteの環境変数（import.meta.env）を使用して管理しました。GitHubにキーが流出しないように配慮しています。

振り返り

今回の開発では、Gitを使用して細かい単位でコミットすることを意識しました。APIキーなどの重要な情報をGitHubに公開しないよう、.gitignoreファイルを正しく設定することを学びました。開発環境（ローカル）と公開環境で環境変数を使い分ける経験は、エンジニアとして非常に大切なステップだと感じました。
また、async/awaitを使ったAPI通信において、正常な場合だけでなく、失敗した場合（try...catch）の処理を書くことの重要性を学びました。
